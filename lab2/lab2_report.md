# Лабораторная работа №2 — Подключение бота к данным

## 1. Описание интеграции

### Какой источник данных выбрали

**SQLite база данных** — встроенная реляционная база данных для хранения результатов тестирования пользователей.

### Почему именно этот

- **Встроенность**: SQLite не требует отдельного сервера, работает локально
- **Асинхронность**: используется `aiosqlite` для неблокирующей работы с БД
- **Масштабируемость**: легко расширять структуру таблиц
- **Надёжность**: ACID-гарантии, транзакции
- **Практичность**: реальный сценарий — сохранение истории пользователей

### Структура данных

```sql
CREATE TABLE results (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER NOT NULL,
    username    TEXT,
    first_name  TEXT,
    mode        TEXT NOT NULL,   -- 'general', 'topic', 'learning'
    topic       TEXT,            -- NULL для общего теста
    score       INTEGER NOT NULL,
    total       INTEGER NOT NULL,
    date        TEXT NOT NULL
)
```

**Поля:**
- `user_id` — Telegram ID пользователя (уникальный идентификатор)
- `username` — @username в Telegram (может отсутствовать)
- `first_name` — имя пользователя
- `mode` — режим теста (общий/тематический/обучение)
- `topic` — ключ темы (только для тематического теста)
- `score` — количество правильных ответов
- `total` — всего вопросов в тесте
- `date` — дата и время прохождения теста

## 3. Реализация

### Как работает интеграция

**Архитектура:**

```
bot.py (основная логика)
    ↓
database.py (работа с БД)
    ↓
finance_bot.db (SQLite база)
```

**Процесс:**

1. **Инициализация** (`init_db`):
   - При запуске бота создаётся таблица `results` (если её нет)
   - Используется асинхронное подключение к SQLite

2. **Сохранение результата** (`save_result`):
   - После завершения теста вызывается функция сохранения
   - Данные вставляются в таблицу с текущей датой/временем

3. **Получение истории** (`get_user_results`):
   - Возвращает последние 10 результатов пользователя
   - Отсортированы по дате (новые сверху)

4. **Статистика** (`get_user_stats`):
   - Подсчитывает общее количество тестов
   - Вычисляет средний процент правильных ответов
   - Используется для отображения в меню "Мои результаты"

### Ключевые фрагменты кода

**Инициализация БД:**
```python
async def init_db() -> None:
    """Инициализация базы данных: создание таблиц при первом запуске."""
    async with aiosqlite.connect(DB_PATH) as db:
        await db.execute("""
            CREATE TABLE IF NOT EXISTS results (
                id          INTEGER PRIMARY KEY AUTOINCREMENT,
                user_id     INTEGER NOT NULL,
                username    TEXT,
                first_name  TEXT,
                mode        TEXT NOT NULL,
                topic       TEXT,
                score       INTEGER NOT NULL,
                total       INTEGER NOT NULL,
                date        TEXT NOT NULL
            )
        """)
        await db.commit()
```

**Сохранение результата:**
```python
async def save_result(
    user_id: int,
    username: str | None,
    first_name: str | None,
    mode: str,
    topic: str | None,
    score: int,
    total: int,
) -> None:
    """Сохраняет результат прохождения теста."""
    date_str = datetime.now().strftime("%d.%m.%Y %H:%M")
    async with aiosqlite.connect(DB_PATH) as db:
        await db.execute(
            """
            INSERT INTO results (user_id, username, first_name, mode, topic, score, total, date)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?)
            """,
            (user_id, username, first_name, mode, topic, score, total, date_str),
        )
        await db.commit()
```

**Получение статистики:**
```python
async def get_user_stats(user_id: int) -> dict:
    """Возвращает агрегированную статистику пользователя."""
    async with aiosqlite.connect(DB_PATH) as db:
        async with db.execute(
            """
            SELECT
                COUNT(*)                        AS total_tests,
                SUM(score)                      AS total_correct,
                SUM(total)                      AS total_questions,
                ROUND(AVG(CAST(score AS REAL) / total * 100), 1) AS avg_percent
            FROM results
            WHERE user_id = ?
            """,
            (user_id,),
        ) as cursor:
            row = await cursor.fetchone()
            if row is None or row[0] == 0:
                return {}
            return {
                "total_tests": row[0],
                "total_correct": row[1],
                "total_questions": row[2],
                "avg_percent": row[3],
            }
```

## 5. Трудности и решения

### Проблема 1: Асинхронность и блокировка event loop

**Проблема**: Обычная синхронная работа с SQLite блокирует event loop бота, что замораживает обработку других сообщений.

**Решение**: Использована библиотека `aiosqlite`, которая позволяет работать с БД асинхронно через `async with` и `await`:
```python
async with aiosqlite.connect(DB_PATH) as db:
    await db.execute(...)
    await db.commit()
```

### Проблема 2: Обработка ошибок при работе с БД

**Проблема**: Если БД недоступна или произойдёт ошибка при вставке, бот может упасть.

**Решение**: Добавлена обработка исключений в `finish_quiz`:
```python
try:
    await save_result(...)
except Exception as e:
    logger.error(f"Ошибка сохранения результата: {e}")
```

### Проблема 3: Пустые результаты при первом запуске

**Проблема**: Если пользователь ещё не прошёл ни одного теста, функция возвращает пустой список.

**Решение**: Добавлена проверка в `show_results`:
```python
if not results:
    text = "У вас пока нет пройденных тестов.\nНажмите «Пройти тест», чтобы начать!"
```

### Проблема 4: Форматирование даты и времени

**Проблема**: Дата сохранялась в разных форматах, что затрудняло сортировку.

**Решение**: Использован единый формат `"%d.%m.%Y %H:%M"` для всех записей:
```python
date_str = datetime.now().strftime("%d.%m.%Y %H:%M")
```
