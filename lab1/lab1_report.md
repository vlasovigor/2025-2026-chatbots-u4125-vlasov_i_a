# Лабораторная работа №1 — Telegram-бот для проверки финансовой грамотности

## 1. Описание задачи

### Какую проблему решает бот

Финансовая грамотность населения России остаётся на низком уровне: многие люди не понимают, как работают кредиты, вклады, инфляция, не умеют планировать бюджет и становятся жертвами финансовых мошенников. При этом традиционные форматы обучения (книги, курсы) требуют значительных временных затрат и не дают мгновенной обратной связи.

Telegram-бот решает эту проблему, предоставляя интерактивный инструмент для самопроверки и обучения в формате теста прямо в мессенджере — без регистрации, без установки приложений, в любое удобное время.

### Почему выбрана именно эта задача

- **Актуальность**: финансовая грамотность — одна из ключевых компетенций современного человека
- **Образовательная ценность**: бот не просто проверяет знания, но и объясняет ошибки
- **Демонстративность**: хорошо подходит для демонстрации возможностей Telegram Bot API (кнопки, состояния, база данных)
- **Масштабируемость**: базу вопросов легко расширять

## 6. Трудности и решения

### Проблема 1: Несовместимость версии Python

**Проблема**: на машине установлен Python 3.8, а `python-telegram-bot==21.9` требует Python 3.9+. При попытке установки через `pip3` зависимости устанавливались в Python 3.9 (от Homebrew), тогда как `python3` указывал на Python 3.8.

**Решение**: понижена версия библиотеки до `python-telegram-bot==21.6` — последней, поддерживающей Python 3.8. Также добавлен `from __future__ import annotations` в `bot.py` и `database.py` для поддержки современного синтаксиса аннотаций типов (`str | None`, `list[dict]`) в Python 3.8.

### Проблема 2: Обрезание текста на кнопках

**Проблема**: Telegram обрезает текст кнопок длиннее ~64 символов, из-за чего варианты ответа отображались неполностью.

**Решение**: варианты ответа перенесены в текст сообщения с вопросом (формат: `*А*. Текст варианта`), а кнопки содержат только буквы А/Б/В/Г в одну строку. Это полностью решает проблему и улучшает читаемость.

### Проблема 3: Переменная окружения с другим именем

**Проблема**: в существующем файле `.env` токен хранился под именем `TELEGRAM_BOT_TOKEN`, а бот ожидал `BOT_TOKEN`.

**Решение**: добавлена поддержка обоих имён:
```python
BOT_TOKEN = os.getenv("BOT_TOKEN") or os.getenv("TELEGRAM_BOT_TOKEN", "")
```
