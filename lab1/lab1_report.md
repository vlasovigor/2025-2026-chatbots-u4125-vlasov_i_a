# Лабораторная работа №1 — Telegram-бот для проверки финансовой грамотности

## 1. Описание задачи

### Какую проблему решает бот

Финансовая грамотность населения России остаётся на низком уровне: многие люди не понимают, как работают кредиты, вклады, инфляция, не умеют планировать бюджет и становятся жертвами финансовых мошенников. При этом традиционные форматы обучения (книги, курсы) требуют значительных временных затрат и не дают мгновенной обратной связи.

Telegram-бот решает эту проблему, предоставляя интерактивный инструмент для самопроверки и обучения в формате теста прямо в мессенджере — без регистрации, без установки приложений, в любое удобное время.

### Почему выбрана именно эта задача

- **Актуальность**: финансовая грамотность — одна из ключевых компетенций современного человека
- **Образовательная ценность**: бот не просто проверяет знания, но и объясняет ошибки
- **Демонстративность**: хорошо подходит для демонстрации возможностей Telegram Bot API (кнопки, состояния, база данных)
- **Масштабируемость**: базу вопросов легко расширять

## 2. Промпт для LLM

### Исходный промпт

```
Создай Telegram-бота на Python с использованием библиотеки python-telegram-bot.

Функционал бота:
Бот проверяет знания пользователя по темам финансовой грамотности:
- бюджет, кредиты, вклады, инфляция, налоги, инвестиции, мошенничество, финансовая безопасность

Пользователь проходит тест, а бот:
- задаёт вопросы
- принимает ответы
- считает баллы
- показывает результат
- объясняет ошибки
- даёт рекомендации по слабым темам

Режимы:
1. Общий тест — 10 случайных вопросов, результат в конце
2. Тематический тест — 5 вопросов по выбранной теме
3. Режим обучения — после каждого ответа объяснение

Требования:
- Бот должен быть простым и понятным
- Код должен быть хорошо прокомментирован
- Использовать SQLite для хранения результатов
- Добавить обработку ошибок

Создай: bot.py, requirements.txt, README.md, .env.example
```

### Итерации и улучшения

**Итерация 1** — уточнение расположения файлов:
```
Бот должен быть в корневой папке проекта, а не в lab4.
В боте ещё должны быть кнопки «просмотреть прошлые результаты» 
и отдельная кнопка «пройти тест» в главном меню.
```

**Итерация 2** — исправление отображения вариантов ответа:
```
В боте во время вопросов варианты ответов на кнопках отображаются 
не полным текстом. Исправь это.
```
*Решение*: варианты ответа перенесены в текст вопроса (А/Б/В/Г + полный текст), а кнопки содержат только буквы — это обходит ограничение Telegram на длину текста кнопки (~64 символа).

### Финальный промпт (итоговый)

Финальная версия бота создана через последовательные итерации с уточнениями по структуре проекта, функционалу главного меню и отображению вариантов ответа. Код разбит на три модуля: `bot.py` (логика), `questions.py` (база вопросов), `database.py` (работа с SQLite).

## 3. Стек технологий

| Библиотека | Версия | Назначение |
|---|---|---|
| `python-telegram-bot` | 21.6 | Работа с Telegram Bot API в асинхронном режиме |
| `aiosqlite` | 0.20.0 | Асинхронная работа с SQLite без блокировки event loop |
| `python-dotenv` | 1.0.1 | Загрузка токена из файла `.env` |

### Почему выбраны именно эти библиотеки

- **python-telegram-bot v20+**: современная async-архитектура, встроенный `ConversationHandler` для управления состояниями диалога, активное сообщество и хорошая документация
- **aiosqlite**: позволяет работать с SQLite асинхронно, не блокируя обработку других сообщений
- **python-dotenv**: стандартный способ хранения секретов (токен бота) вне кода

## 6. Трудности и решения

### Проблема 1: Несовместимость версии Python

**Проблема**: на машине установлен Python 3.8, а `python-telegram-bot==21.9` требует Python 3.9+. При попытке установки через `pip3` зависимости устанавливались в Python 3.9 (от Homebrew), тогда как `python3` указывал на Python 3.8.

**Решение**: понижена версия библиотеки до `python-telegram-bot==21.6` — последней, поддерживающей Python 3.8. Также добавлен `from __future__ import annotations` в `bot.py` и `database.py` для поддержки современного синтаксиса аннотаций типов (`str | None`, `list[dict]`) в Python 3.8.

### Проблема 2: Обрезание текста на кнопках

**Проблема**: Telegram обрезает текст кнопок длиннее ~64 символов, из-за чего варианты ответа отображались неполностью.

**Решение**: варианты ответа перенесены в текст сообщения с вопросом (формат: `*А*. Текст варианта`), а кнопки содержат только буквы А/Б/В/Г в одну строку. Это полностью решает проблему и улучшает читаемость.

### Проблема 3: Переменная окружения с другим именем

**Проблема**: в существующем файле `.env` токен хранился под именем `TELEGRAM_BOT_TOKEN`, а бот ожидал `BOT_TOKEN`.

**Решение**: добавлена поддержка обоих имён:
```python
BOT_TOKEN = os.getenv("BOT_TOKEN") or os.getenv("TELEGRAM_BOT_TOKEN", "")
```
