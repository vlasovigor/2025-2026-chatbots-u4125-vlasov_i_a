# Лабораторная работа 3: Деплой Telegram-бота для проверки финансовой грамотности

В этой лабораторной работе был развернут Telegram-бот для проверки уровня финансовой грамотности пользователей. Бот был успешно развернут на удаленном сервере с использованием Docker и Docker Compose, что позволило сделать его доступным для реальных пользователей 24/7.

## Шаг 3: Деплой

### 3.1 Создание Dockerfile

```dockerfile
FROM python:3.11-slim

WORKDIR /app

RUN apt-get update && apt-get install -y gcc && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

RUN mkdir -p /app/logs /app/data

ENV PYTHONUNBUFFERED=1

CMD ["python", "bot.py"]
```

### 3.2 Создание docker-compose.prod.yml

```yaml
version: '3.3'

services:
  finance-bot:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: finance-bot-prod
    restart: always
    
    environment:
      - BOT_TOKEN=${BOT_TOKEN}
      - PYTHONUNBUFFERED=1
    
    volumes:
      - /data/finance-bot/db:/app/data
      - /data/finance-bot/logs:/app/logs
    
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
    
    healthcheck:
      test: ["CMD", "python", "-c", "import os; exit(0 if os.path.exists('/app/data') else 1)"]
      interval: 60s
      timeout: 10s
      retries: 5
    
    logging:
      driver: "json-file"
      options:
        max-size: "50m"
        max-file: "10"
        labels: "service=finance-bot"
    
    networks:
      - finance-bot-network
    
    security_opt:
      - no-new-privileges:true

networks:
  finance-bot-network:
    driver: bridge

volumes:
  bot-data:
    driver: local
```

### 3.3 Создание скрипта деплоя (deploy.sh)

Создан автоматизированный скрипт для развертывания бота на удаленном сервере:

```bash
./deploy.sh <server-ip> <bot-token> [username] [ssh-key]
```

**Функции скрипта:**
- ✅ Проверка подключения к серверу
- ✅ Создание необходимых директорий
- ✅ Копирование файлов на сервер
- ✅ Создание .env файла
- ✅ Проверка Docker
- ✅ Автоматическая установка docker-compose
- ✅ Запуск контейнера
- ✅ Проверка статуса
- ✅ Просмотр логов

### 3.4 Процесс деплоя

```bash
# 1. Подготовка
chmod +x deploy.sh

# 2. Запуск деплоя
./deploy.sh 192.168.1.100 YOUR_BOT_TOKEN root ~/.ssh/id_rsa

# 3. Проверка статуса
ssh root@192.168.1.100 "cd /opt/finance-bot && docker-compose -f docker-compose.prod.yml ps"

# 4. Просмотр ло��ов
ssh root@192.168.1.100 "cd /opt/finance-bot && docker-compose -f docker-compose.prod.yml logs -f finance-bot"
```

### 3.5 Проблемы и решения

| Проблема | Решение |
|----------|---------|
| `unknown shorthand flag: 'f' in -f` | Использовать правильный флаг для версии docker-compose |
| `TELEGRAM_BOT_TOKEN variable is not set` | Исправить переменную окружения на BOT_TOKEN |
| `Version in docker-compose.prod.yml is unsupported` | Изменить версию с 3.8 на 3.3 |
| `start_period not allowed in healthcheck` | Удалить start_period (не поддерживается в 3.3) |
| `Range of CPUs is from 0.01 to 1.00` | Уменьшить лимиты CPU с 2 на 0.5 |
| `unable to open database file` | Исправить пути volume для БД |
| `docker-compose не установлен` | Добавить автоматическую установку в скрипт |
